# OMG Specifications — Signpost (download pointers only)

**This is a signpost, not a knowledge pack.** It carries **no OMG content** — no
reproduced text, no synthesised summaries. The OMG Specification License permits only
informational, unmodified, non-network-posted use, so OMG specs cannot be reconstituted
into a redistributable knowledge pack (see this repo's `docs/SOURCE-VETTING.md`). What
this skill does: tell you which spec you want and send you to OMG to get it.

## When to use
You're modelling and reach for an OMG language/framework — UML, SysML, BPMN, UAF, etc.
This points you at the authoritative source; read it from OMG, don't expect it summarised here.

**Prerequisites:** none — plain Markdown.

## How to use
Find your spec below, then download the current version from OMG. Every OMG spec lives at
`https://www.omg.org/spec/<NAME>` (e.g. `/UML`, `/SysML`, `/BPMN`), which always links the
latest formal version. Catalogue of all OMG specs: `https://www.omg.org/spec/`.

## The specs (what each is for — facts only)

| Spec | What it is | Get it at |
|------|-----------|-----------|
| **UML** | Unified Modeling Language — general-purpose software/systems modelling | omg.org/spec/UML |
| **SysML** | Systems Modeling Language — systems engineering (a UML profile; v1.x) | omg.org/spec/SysML |
| **SysML v2 / KerML** | Next-generation SysML and its kernel language | omg.org/spec/SysML (v2 listings) |
| **BPMN** | Business Process Model and Notation — process modelling | omg.org/spec/BPMN |
| **UAF** | Unified Architecture Framework — enterprise/defence architecture | omg.org/spec/UAF |
| **SoaML** | Service oriented architecture Modeling Language | omg.org/spec/SoaML |
| **MOF** | Meta Object Facility — metamodelling foundation | omg.org/spec/MOF |
| **OCL** | Object Constraint Language — constraints on models | omg.org/spec/OCL |
| **XMI** | XML Metadata Interchange — model serialisation/exchange | omg.org/spec/XMI |
| **DDS** | Data Distribution Service — real-time pub/sub data | omg.org/spec/DDS |

## Why not a real pack?
The OMG public grant says the spec "will not be copied or posted on any network computer …
and no modifications are made." A hosted, transformed pack would breach both, so the whole
OMG `formal/` corpus is on this repo's Excluded list. This signpost stays clean by holding
only spec **names, purposes, and the OMG URL** — facts and a citation, never the source text.
