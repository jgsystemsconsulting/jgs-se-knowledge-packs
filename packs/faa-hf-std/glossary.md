# FAA HF-STD-001B Glossary

Key terms from the FAA Human Factors Design Standard, alphabetical, each with the chapter(s) where it is used. Definitions are paraphrased from the standard's controlled vocabulary (Section 3) and from the section that operationalizes each term.

- **Adaptive automation** — automation that re-divides tasks between the operator and the machine in real time as situational demand shifts, deliberately keeping the operator an active controller rather than a passive watcher. (ch01, ch02)
- **Advisory** — a signal class denoting a safe or normal configuration, or routine information; the lowest-urgency member of the alert family. (ch01, ch05)
- **Alarm filtering** — suppressing alarms outright when they carry no current operational meaning, so they never reach the operator. Distinct from suppression. (ch01, ch05)
- **Alarm suppression** — hiding alarm messages while keeping them retrievable on request, used for the predictable downstream cascade of a primary event. (ch01, ch05)
- **Alert** — a signal demanding immediate action or flagging a significant update; in the controlled glossary, distinct from caution, advisory, and warning. (ch01, ch05)
- **Anthropometry** — the body-measurement statistics (stature, reach, strength, clearance) a design must accommodate across its user population. (ch01, ch08, ch09)
- **Authentication** — confirming that a station, originator, or user is eligible to reach specific information; guards against fraudulent entry. (ch09)
- **Authorization** — granting a user or group the right to use a program, process, or body of information. (ch09)
- **Automation** — a device or system that independently performs a function a person once did; in this standard, always intended to support rather than displace the operator. (ch02)
- **Automation bias** — over-relying on an automated aid heuristically instead of actively gathering and weighing the available evidence. (ch02)
- **Caution** — a signal class for a condition that needs attention but not immediate action, or that warns of possible equipment damage. (ch01, ch05)
- **Comfort zone** — the band of thermal conditions in which comfort is attainable, shifting with work rate, clothing, and acclimatization. (ch09)
- **Complacency** — the reduced vigilance and overlooked errors that follow from consistently reliable automation. (ch02)
- **Component** — a subdivision of a unit that a user treats as one object but that can be broken down further (e.g. a populated board). (ch03)
- **Control automation** — an automation type that executes actions or control tasks with some autonomy, under bounded authority and guaranteed override for safety-critical functions. (ch02)
- **Dead-man control** — a control that drives the system to a safe state when the operator's force or input is removed, used where operator incapacity could be critical. (ch04)
- **Decision aid** — an automated system that supports a human decision by narrowing alternatives or suggesting a preferred option; it assists, never replaces, the decision maker. (ch02)
- **Design-limits approach** — sizing a design from population statistics so a chosen portion of the user population is accommodated. (ch01, ch09)
- **Direct manipulation** — an interaction style in which the user acts directly on concrete on-screen objects (icons, buttons, symbols); suits casual or untrained users. (ch07)
- **Effective temperature (ET)** — an empirical index combining dry-bulb, wet-bulb, air velocity, and clothing to express thermal sensation. (ch09)
- **Effectiveness** — producing the desired result, as distinct from doing so with minimum waste. (ch01)
- **Efficiency** — producing a result with minimum waste, typically of time. (ch01)
- **Error resistance** — making an error hard to commit in the first place, through simplicity and clear information. (ch01, ch02)
- **Error tolerance** — accepting that some errors will occur and limiting their consequences, e.g. by adding monitoring. (ch01, ch02)
- **Fault management** — the user-facing handling of failures: detection, diagnosis, prognosis, and compensation. (ch02)
- **Filtering vs suppression** — see *Alarm filtering* and *Alarm suppression*; a glossary distinction the alarm rules enforce. (ch01, ch05)
- **Function allocation** — deciding which functions go to the machine and which to the person; evaluated across fully manual, partially automated, fully automated, and adaptive schemes. (ch02)
- **Hazardous condition** — energy or a substance likely to cause death or injury (shock, force, radiation, flame, poison, corrosion, biological/chemical effects). (ch03, ch09)
- **Hotspot** — the single precise active point of a pointer that marks where an operation will occur; it must not shift when the pointer changes shape. (ch08)
- **Hysteresis** — the lag a body shows in reacting to changing forces; relevant to joystick dead-band behaviour. (ch08)
- **Identification** — recognizing a previously listed user name; the first of the identification/authentication/authorization triad. (ch09)
- **Information automation** — an automation type covering data acquisition and integration (filtering, transforming, integrity checks), aimed at sustaining situation awareness. (ch02)
- **Information density** — the amount of information per unit of screen area, to be minimized to what the task needs now. (ch06)
- **Input focus** — the state in which exactly one window, and usually one object within it, receives pointer and keyboard input. (ch07)
- **Interactive electronic technical manual (IETM)** — electronic documentation offering paired novice/expert detail levels, troubleshooting logic, and consistent navigation. (ch09)
- **Interlock** — a device that disables an internal hazard when a cover or case is opened; never a substitute for lockout/tagout. (ch03, ch09)
- **Keyguard** — a keyboard cover with holes over the keys, an accessibility aid for users with moderate motor disabilities. (ch08)
- **Lockout / tagout** — mechanically disabling a control in its safe position (lockout) and attaching a tag at the hazard (tagout); used together, per OSHA. (ch03, ch09)
- **Luminance** — the measured brightness of a display surface; bounded for uniformity, contrast, and dimming. (ch04, ch06)
- **Mental model** — an individual's understanding of the processes underlying a system's operation; a design target for monitoring, diagnosis, and prediction. (ch02)
- **Mnemonic vs accelerator** — a mnemonic is an underlined letter typed while a menu is shown; an accelerator is a key combination that fires an option without showing the menu. (ch07)
- **Modality (window)** — a modal window must be dealt with before any other; a modeless window does not block other interaction. (ch07)
- **Module** — two or more interconnected parts forming one physical-and-functional entity; the single functionality that makes something a module aids replacement. (ch01, ch03)
- **Order of precedence** — the rule that, on conflict, the standard's own text governs over a cited reference, but never over applicable law or regulation. (ch01, ch06)
- **Part** — an item that cannot normally be broken down further without destroying its use (fuse, resistor, capacitor). (ch03)
- **Percentile (anthropometric)** — a population statistic used to set clearance, reach, and adjustability dimensions (e.g. 95th-male clearance, 5th-female reach). (ch01, ch08, ch09)
- **Pointing device** — a non-keyboard device for rapid screen navigation and object selection (mouse, trackball, joystick, stylus). (ch08)
- **Salience window** — the band a warning signal must hit: loud and distinct enough to be detected, but not so loud or startling as to block the right response. (ch05)
- **Screen density** — the ratio of filled to blank character positions in a text display, kept below roughly 60%. (ch06)
- **Security safeguards** — protective measures and controls (procedural, physical, hardware/software) prescribed to meet a system's security requirements, treated as a subsystem. (ch09)
- **Situation awareness** — the operator's current understanding of system state, eroded by passive monitoring and excessive automation. (ch02)
- **Trust calibration** — building trust in automation that matches its actual reliability under given conditions; both over-trust and under-trust are failure states. (ch02)
- **Unit of equipment** — an assemblage of items packaged as a single hardware package (a computer, keyboard, display, or transmitter). (ch03)
- **Unitization** — packaging equipment into physically and functionally distinct, easily replaceable units. (ch03)
- **User-centered perspective** — focusing on the end user's needs throughout design, acquisition, and development. (ch01)
- **Vigilance decrement** — the declining ability to sustain attention during monitoring, driven by the cognitive load of vigilance rather than boredom. (ch02)
- **Warning** — a signal class for a hazardous condition needing immediate action to prevent loss of life, equipment damage, or service interruption. (ch01, ch05)
- **Workplace** — the broader area, room, or establishment where work is done. (ch08)
- **Workstation** — a place designed for a specific task from which work is conducted (desk, console, terminal, repair bench). (ch08)
